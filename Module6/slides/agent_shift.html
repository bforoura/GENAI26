Chapter 6: Structural Reliability in Multi-Agent Systems

2023: Mega-Prompting Unreliable

Reliability is requested via linguistic pressure. A single point of failure.
### SYSTEM PROMPT ### "You are a world-class board-certified cardiologist with 30 years of experience in emergency triage. You are currently working in a high-stakes environment where accuracy is 100% mandatory. CRITICAL SAFETY PROTOCOL: Scan the user input for 'Red Flags' (chest pain, radiating jaw pain, shortness of breath, diaphoresis). If ANY of these are present, you must ignore all other instructions and return a Triage-1 Emergency JSON. ROLE CONSTRAINTS: 1. Act with extreme empathy but remain clinical. 2. Cross-reference the user input against the provided medical history. 3. If history includes 'Asthma' but symptoms are 'Tightness', you must weigh Cardiac risk 70% and Respiratory risk 30%. 4. Take a deep breath. Think through the clinical path step-by-step. 5. This is mission-critical; a failure in formatting will lead to downstream system failure. OUTPUT REQUIREMENTS: - Return ONLY valid JSON. - Schema: {'level': int, 'risk_factor': string, 'logic': string}. - Do NOT include triple backticks, markdown, or conversational text. - If you output anything other than JSON, a penalty will be applied. - Your performance is vital to the survival of this clinic." ### INPUT ### "My chest is tight and I have a history of heart issues."

2026: Agentic Graph Deterministic

Reliability is a structural property managed by the state machine routing.
### AGENT 1: SAFETY SCANNER ### "Search for: 'chest pain', 'shortness of breath'." ### AGENT 2: HISTORY RETRIEVER ### "Query EHR database for past cardiac events." ### AGENT 3: TRIAGE COORDINATOR ### "Combine safety report and history to assign priority." ### STATE GRAPH (LangGraph) ### - Start -> Safety_Node - Safety_Node -> IF 'Critical' THEN ER_Alert ELSE History_Node - History_Node -> Triage_Node - Triage_Node -> End ### OBSERVATION ### Reliability is managed by the graph's edges and conditional logic, not by begging the model to perform well.
