Parallelism Architecture

Task Latency vs. Data Throughput

CPU

Serial Processor

"Do one complex thing very fast"

CORE 0
CORE 1
→

Latency Optimized: Built with massive caches and complex branch prediction to finish individual tasks quickly.

→

MIMD: (Multiple Instruction, Multiple Data). Each core can do something totally different (e.g., Core 0 handles WiFi, Core 1 handles Chrome).

GPU

Parallel Processor

"Do thousands of simple things at once"

→

Throughput Optimized: Sacrifices complex logic for massive volume. Ideal for the matrix multiplication inside LLMs.

→

SIMD: (Single Instruction, Multiple Data). Thousands of cores do the *exact* same math on different chunks of the model weights.

Best For OS, System Logic, I/O
Best For Matrix Math, Video, AI
